## Key decisions
- Keep the derived parquet bundle detailed and apply the `10-item` rule in the Streamlit presentation layer, not by globally truncating the underlying aggregates.
- Treat sidebar selectors as part of the same contract: each dropdown now keeps the `All ...` option plus the top 9 real categories by posting volume.
- Align CLI defaults with the same user-facing rule by moving `--top-markets-per-province` and `--skills-top-k` defaults to `10`.
- Preserve lazy CLI imports while restoring test patchability through lightweight module-level wrappers in `src/jobads_dashboard/cli.py`.
- Treat `All Canada` / `All occupations` / `All industries` as wildcards for the Skills tab rather than expecting materialized sentinel rows in `monthly_skills_topk`.
- Treat the redesign job as complete once a dedicated inspection subagent can no longer find a meaningful issue in the agreed checklist.
- Treat province- and industry-scoped compensation views as first-class surfaces that must read from the fully filtered wage cube, not from the national-only occupation wage extract.
- Treat the collapsed mobile sidebar as a true off-canvas surface: it must not reserve layout space or push the main dashboard content down the page.
- Treat Plotly modebar controls as non-essential chrome for this audience-facing dashboard surface; keep them hidden unless a future interaction need clearly justifies exposing them again.
- Treat partial derived bundles as an operator-facing error state, not an internal traceback: validate required files up front, name the missing parquet surfaces, and tell the user to run `refresh` then `validate`.
- Treat ngrok deployment as part of the done condition for page/dashboard UI work in this repo, not as an optional afterthought.
- Treat the currently live ngrok URL as canonical while the tunnel remains active; do not create a replacement URL casually.
