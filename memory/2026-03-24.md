## Key decisions
- `jobads-dashboard` is the canonical dashboard repo.
- `../jobads-data/main` remains the upstream data source, not the implementation home.
- Version 1 should build local aggregates under `data/derived/labor_market_dashboard_v1/` and local code under `src/jobads_dashboard/`.
- The dashboard remains descriptive and economics-driven, with no dependency on `ai_labor`.
- The first implementation uses DuckDB-backed aggregate refresh plus a Streamlit app with single-select global filters and local-only runtime reads.
- The app launch surface is now `streamlit_app.py`, while the operator CLI is `jobads-dashboard`.

## Risks
- Git history is unavailable for this root at bootstrap time.
- Upstream `jobads-data/main` still carries the known `2025` fetch-provenance caveat.
- Several desirable descriptive fields remain sparse or historically unstable.
- The skills top-k aggregate is materially slower than the rest of the refresh pipeline.
- Browser automation via the provided Playwright wrapper was not the reliable verification path in this environment, so UI verification used Streamlit's app-testing harness plus live server health checks instead.
